OVERVIEW


In this in-depth Case study, I designed and developed a website portfolio intended for myself wherein I can showcase my capabilities and projects I made. This case study presents all the design processes and development I've done in a step-by-step procedure from visualizing to a finished deployed website portfolio. I also researched about the things that should on what a website portfolio should have, the Do’s and Don’ts in creating a website portfolio.





DESIGN & DEVELOPMENT PROCESS


The design & development process of my website portfolio includes several levels to take to accomplish and produce a solid website portfolio. Starting with project goal, which identifies the main purpose of the creation of website portfolio. Followed by diagram, which includes the sitemap. Followed by wireframe, which includes the low-fidelity and high-fidelity. Followed by user interface which includes all the components I used for the design as well as creating the prototype. After design I proceeded to the development where I will turn the design into actual codes. Lastly, the deployment of the website over the internet.

Project Goal


Goal Identification


As a UI/UX designer / Web Designer it is vital to create my own website portfolio to showcase my skills and work to potential employers or clients. Having a personal website allows me to present my projects in an organized and visually appealing manner, demonstrating my design expertise and problem-solving abilities. It serves as a centralized hub where I can exhibit a diverse range of projects, highlight my design process, and provide contact information for professional inquiries. The project goal of this case study is to provide a well-crafted portfolio website that helps me make a strong impression and stand out in the competitive field of design.

USER INTERFACE


Now that I've completed low-fidelity and high-fidelity wireframe, I've started working on the user interface, which will illustrate how the website portfolio will look like. Important visual elements of the User Interface that improve aesthetics and recognition of the brand are colors, typography, icons, and images. Users can interact realistically with the design through the use of interaction components like buttons, forms, and navigation that appear in user interfaces.
View User Interfaces on Figma



Colors & Typography


The color palette of #121314, #fdbd16, and #0c0c0c is a solid choice for a website portfolio due to its effective use of contrast, professionalism, and modern aesthetic. The dark background colors (#121314 and #0c0c0c) convey sophistication and provide a visually appealing backdrop for content, enhancing readability and guiding users through the portfolio. The bright, saturated color (#fdbd16) adds a vibrant accent, highlighting key elements and contributing to a memorable and cohesive design. This combination creates a contemporary feel and, when applied consistently, establishes a unified visual identity that is both modern and professional, making it suitable for showcasing a diverse range of work.

Open Sans is a versatile and legible font that offers excellent readability across various screen sizes, making it a solid choice for web design. Its clean, modern aesthetic and wide range of font weights provide flexibility, ensuring a professional and accessible appearance for both headers and body text on a website.
